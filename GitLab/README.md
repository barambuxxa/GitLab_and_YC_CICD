Тут будет документация по работе в GitLab. Дублирование проекта, который уже есть в GitLab.

У нас уже развёрнута VM с GitLab. Заходим на неё и создаём проект.

Для запуска CICD Pipeline, нам необходим runner. Мы будем использовать нашу VM для запуска pipeline. Ход настройки:
- # Добавляем официальный репозиторий GitLab Runner
- curl -L "https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.deb.sh" | sudo bash

- # Устанавливаем сам runner
- sudo apt-get install gitlab-runner

Теперь нужно "познакомить" Runner с нашим GitLab-сервером. Для этого понадобятся URL вашего сервера и токен регистрации.
Где взять токен:
1.	Зайдите в ваш проект на GitLab
2.	Перейдите Settings → CI/CD → Runners
3.	Найдите раздел Project runners (или Create project runner)
4.	Скопируйте токен (обычно начинается с GR1348941)
Вас спросят несколько параметров:
1.	URL вашего GitLab: введите http://<IP-вашего-сервера> (или домен)
2.	Токен: вставьте скопированный токен
3.	Описание: например, "my-first-runner"
4.	Теги (tags): ВАЖНО! Если вы не хотите возиться с тегами сейчас — просто оставьте пустым и нажмите Enter
5.	Executor (исполнитель): для начала выберите shell (самый простой, выполняет команды прямо на сервере)
